Overview

The Prowler is an article aggregator application for publications on Princeton’s campus. The Prowler offers a beautiful interface for easy reading and searching of articles across multiple publications. Articles are updated by the hour, so you'll never miss out on the latest controversy. The user interface dynamic and able to easily expand to support new publications, without any extra installations or updates.

Database Access

Anyone can use the backend for research or data analysis. We allow you to download and query for articles in JSON format, making textual analysis across Princeton publications a reality. Our full backend API can be found here.

Design

Our implementation consists of the three standard interfaces: a user interface, a web crawler, and a database. The user interface compiles to both IOS and Android devices, the web crawler is written in python, and the database is a standard MySQL database. This general framework makes “The Prowler” extendable to other publications (not just the Prince) as our UI can dynamically load all publications. The web scraper is easily extendable to inject new publications and them them to the hourly scrape.

User Interface

The user interface has to be dynamic as the data being supplied to it will change daily - also if we add publications, it should be robust enough to accept that change. Our baseline is a scrolling feed with articles, the next step is a main page organized by section, and then a fully flushed recommendation system (with user login).

Web Service

Each communication with the database is handled through a web service python script running on an online server. These requests will be handled with an HTTP protocol and JSON format data.

Web Scraper

The web scraper is written in python. It lives on an Amazon EC2 instance and pulls new articles hourly and then posts them to the database, while making sure that duplicates do not occur. It is easily extendable to scrape new publications.

Database

Database: The publication data is stored consistently between sessions in a MySQL database which we can use with Heroku but also locally for testing. There is a table for publication data (articles), one for user data, and one for user metadata associated with each article (comments etc.).
